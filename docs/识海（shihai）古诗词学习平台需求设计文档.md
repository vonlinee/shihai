# 识海（shihai）古诗词学习平台需求设计文档

# 一、文档概述

## 1.1 文档目的

本文档针对“识海（shihai）古诗词学习平台”进行全流程、全方位的需求设计说明，明确系统核心需求、功能边界、角色职责、业务流程及非功能要求，作为项目需求确认、开发落地、测试校验的核心依据，确保需求传递准确，实现系统预期目标，满足用户学习、互动、纠错及管理员运维管理的核心诉求。

## 1.2 文档范围

本文档覆盖识海平台全部核心需求，包括项目概述、数据来源、角色划分、功能需求、系统流程、非功能需求等，重点整合游客匿名评论、多级评论、评论点赞点踩、诗词纠错申请及审核修改流程等新增需求，涵盖管理员端、普通用户端、游客端及细分角色（资源审核、资源修改）的全功能需求，不包含技术架构、数据库设计、系统实现等开发相关细节。

## 1.3 项目概述

识海（shihai）古诗词学习平台是一款基于B/S架构、前后端分离模式开发的线上古诗词学习与交流平台。平台面向普通用户与系统管理员，提供诗词学习、互动交流、资源管理、权限管控、诗词纠错等核心功能，支持游客匿名参与评论互动，兼顾开放性与安全性，打造沉浸式、社区化的古诗词学习环境。

# 二、数据来源

系统诗词及相关资源数据来源多元化，确保资源丰富性、准确性，同时降低数据采集成本，具体来源如下：

1. 开源数据同步：从GitHub开源项目（https://github.com/chinese-poetry/chinese-poetry）批量导入全量唐诗、宋词、古诗文原文、作者信息等基础数据，减少人工录入成本

2. 爬虫采集：通过网络爬虫定向采集诗词赏析、诗人生平介绍、诗词讲解视频链接等扩展资源，丰富平台内容

3. 人工录入：管理员通过后台手动添加、编辑优质诗词资源、问答题目、公告等内容，确保数据质量与唯一性

4. 用户纠错补充：通过用户提交的诗词纠错申请，经审核通过后，由资源修改角色用户更新诗词数据，完善资源准确性

数据导入后，由管理员进行审核、编辑，确保数据准确无误、符合平台定位，再同步至前端展示；用户提交的纠错申请经审核流程通过后，同步更新对应诗词数据。

# 三、系统需求分析

## 3.1 系统总体目标

搭建“管理员—用户—游客”三级安全互动的古诗词线上学习与交流平台，实现诗词资源数字化管理、沉浸式学习、多场景互动、精细化权限管控、诗词纠错优化等核心能力。系统以用户体验为核心，以数据安全为基础，满足诗词爱好者学习、赏析、交流、纠错反馈的需求，同时支撑管理员高效完成系统运维与资源管理，打造专业、便捷、开放、精准的古诗词学习社区。

## 3.2 角色划分

系统分为三个基础角色，权限隔离、功能独立，分别为：管理员（最高权限）、普通用户（注册登录用户）、游客（未登录用户）；新增细分角色：资源审核角色、资源修改角色（均从管理员角色中划分，继承管理员基础权限，专注于纠错审核与资源修改），各角色职责明确，确保系统有序运行。

## 3.3 功能需求分析

### 3.3.1 管理员功能需求（最高权限）

管理员负责全系统的资源管控、用户管理、互动管控与系统运维，核心功能如下：

1. 用户管理：对普通用户账号进行新增、查询、修改、删除操作，管理用户基础信息（账号、姓名、性别、年龄、手机号、身份证、头像等），可禁用违规用户账号；分配资源审核、资源修改角色权限。

2. 诗词资源管理：
        

    - 古诗词佳作朗读管理：新增、修改、删除诗词信息，管理诗词音频、封面图、浏览量、评论等内容，审核诗词释义与赏析内容。

    - 古诗词佳作视频管理：新增、修改、删除诗词视频资源，维护视频链接、封面图、简介、发布时间等信息，确保视频内容合规。

    - 基础数据管理：对朝代、诗人、诗词题材进行统一维护（新增、修改、删除），建立诗词分类体系，便于用户检索与学习。

3. 互动功能管理：
        

    - 诗词问答管理：新增、修改、删除问答题目，维护题目选项、正确答案、解析等内容。

    - 答题记录管理：查看所有用户的答题记录，统计答题数据，支持按用户、题目进行检索。

    - 诗词论坛管理：审核、删除违规论坛帖子及回复，管理论坛内容，维护论坛秩序。

    - 评论管理：查看全平台所有评论及子回复，支持单条删除、批量删除、屏蔽违规评论，可按诗词ID、游客标识、关键词检索评论。

4. 纠错管理：查看全平台所有诗词纠错申请，监控审核进度、投票结果、修改情况，可干预异常纠错流程（如终止违规纠错申请）。

5. 系统公共管理：发布、编辑、删除系统公告，查看并处理用户提交的在线反馈，配置系统基础参数（如评论限流规则、敏感词库、纠错投票规则、消息推送设置）。

6. 个人中心：修改个人登录密码、查看自身操作日志，管理个人账号信息。

### 3.3.2 普通用户功能需求（注册登录用户）

普通用户为系统核心使用者，专注于诗词学习与互动操作，核心功能如下：

1. 账号操作：用户注册（完成信息填写与校验）、账号登录（用户名+密码验证）、身份验证，支持修改个人登录信息。

2. 首页浏览：查看系统推荐的诗词资源、最新公告信息、热门互动内容（热门评论、热门帖子）。

3. 诗词学习：
        

    - 古诗词佳作朗读：查看诗词名称、题材、释义、朝代、诗人、出处等全维度信息，播放诗词音频，支持对诗词进行点赞、点踩、评论、收藏。

    - 古诗词佳作视频：在线观看诗词讲解视频，查看视频详情与简介。

    - 诗词赏析：浏览专业的诗词赏析内容，提升学习体验，加深对诗词的理解。

4. 互动操作：
        

    - 诗词评论：对诗词发表一级评论，对任意评论进行二级回复（楼中楼），可删除自己发布的评论及子回复，查看评论列表并按时间排序。

    - 评论互动：对任意评论、子回复进行点赞、点踩，同一用户对同一评论仅可操作一次，可取消点赞/点踩，实时查看点赞数、点踩数。

    - 诗词问答：查看问答题目、参与在线答题，提交回答内容，查看答题结果与解析。

    - 诗词论坛：发布论坛帖子、查看他人帖子，对帖子进行回复，参与交流讨论。

5. 纠错申请：针对诗词原文、释义、赏析等内容的错误，提交纠错申请，填写纠错内容、纠错依据，查看自己提交的纠错申请进度（待审核、审核中、已通过、已驳回）。

6. 辅助功能：提交在线反馈（问题与建议）、查看系统公告，了解平台最新动态。

7. 个人中心：修改个人基础信息（姓名、性别、年龄、手机号、头像等），查看我的收藏内容、我的发布（帖子、评论）、我的答题记录、我的纠错申请，管理个人答题记录（修改、删除）。

### 3.3.3 游客功能需求（未登录用户）

游客无需注册、登录即可访问平台公共内容，参与基础互动，核心功能如下：

1. 浏览功能：查看首页推荐诗词、系统公告、诗词详情、诗词视频、论坛帖子、评论列表。

2. 评论互动：
        

    - 对诗词发表一级评论，对任意评论进行二级回复，系统自动生成匿名标识（如“匿名用户-xxxx”）。

    - 对任意评论、子回复进行点赞、点踩，同一IP+设备对同一评论仅可操作一次，可取消点赞/点踩。

3. 限制说明：游客无法收藏诗词、发布论坛帖子、参与诗词问答、修改评论，无法查看个人中心相关内容，评论发布受频率限制，**无法提交诗词纠错申请**（需注册登录后操作）。

### 3.3.4 资源审核角色功能需求

资源审核角色由管理员分配权限，专注于诗词纠错申请的审核与投票，核心功能如下：

1. 纠错申请查看：查看系统推送的待审核纠错申请，查看申请详情（纠错诗词、纠错内容、纠错依据、申请人信息）。

2. 投票操作：对每一条待审核纠错申请进行投票（支持/驳回），填写投票意见（可选），同一审核人员对同一纠错申请仅可投票一次，不可修改。

3. 进度查看：查看所有纠错申请的审核进度、投票统计结果，筛选待投票、已投票的纠错申请。

4. 消息接收：接收系统推送的新纠错申请通知、投票结果汇总通知，查看消息历史记录。

5. 个人中心：修改个人登录密码、查看自身审核操作日志，管理个人账号信息。

### 3.3.5 资源修改角色功能需求

资源修改角色由管理员分配权限，专注于经审核通过的纠错申请对应的诗词资源修改，核心功能如下：

1. 修改任务查看：查看系统推送的待修改纠错任务（仅显示投票通过的纠错申请），查看任务详情（纠错诗词、纠错内容、纠错依据、投票结果）。

2. 诗词修改：根据纠错申请内容，修改诗词对应错误信息（原文、释义、赏析等），修改后提交审核（由管理员或指定人员二次校验）。

3. 任务管理：查看所有待修改、已修改、已校验的纠错任务，更新任务进度（待修改、修改中、已提交）。

4. 消息接收：接收系统推送的新修改任务通知、修改任务校验结果通知，查看消息历史记录。

5. 个人中心：修改个人登录密码、查看自身修改操作日志，管理个人账号信息。

## 3.4 系统流程分析

### 3.4.1 用户注册流程

1. 非注册用户访问系统，点击“注册”按钮，进入注册页面；

2. 输入用户名，系统实时校验用户名唯一性，若用户名已存在则提示用户重新输入；

3. 用户名唯一通过后，填写密码、确认密码、姓名、性别、年龄、手机号、身份证号，上传头像；

4. 系统校验密码与确认密码一致性、信息格式合法性（如手机号长度、身份证格式）；

5. 校验通过后，完成注册，系统自动跳转至登录页面；

6. 注册失败，提示具体失败原因（如密码不一致、信息格式错误），用户修改后重新提交。

### 3.4.2 系统登录与权限分配流程

1. 用户/管理员访问登录页面，输入用户名、密码，选择对应角色（普通用户/管理员/资源审核/资源修改）；

2. 系统校验身份信息（用户名、密码、角色匹配度），验证通过后生成登录令牌；

3. 系统根据角色分配对应权限与操作菜单：管理员跳转至后台管理页面，普通用户跳转至前台首页，资源审核角色跳转至纠错审核页面，资源修改角色跳转至诗词修改页面；

4. 安全管控：未登录用户（游客）仅可访问公共页面，无法访问个人中心、后台管理、收藏、答题、纠错申请等核心功能；登录用户仅可访问对应角色权限内的功能，禁止越权操作；

5. 登录失败，提示具体原因（如用户名密码错误、角色选择错误），用户重新输入。

### 3.4.3 评论发布流程（游客/登录用户）

1. 游客/登录用户进入诗词详情页，查看评论列表，点击“发表评论”或“回复”按钮；

2. 输入评论内容，系统校验内容合法性（长度、敏感词、重复内容）；

3. 若为游客：系统自动生成游客唯一标识（IP+浏览器指纹哈希摘要），不存储原始IP与隐私信息；

4. 若为登录用户：系统关联用户ID，自动获取用户名、头像信息；

5. 校验通过后，提交评论，系统插入评论数据，同步更新评论数；

6. 若为回复评论：同步更新父评论的回复数，前端实时渲染评论内容，展示匿名标识或用户名、头像；

7. 校验失败，提示具体原因（如内容包含敏感词、频率过高），用户修改后重新提交。

### 3.4.4 评论点赞/点踩流程

1. 游客/登录用户点击某条评论/子回复的“点赞”或“点踩”按钮；

2. 系统根据用户身份进行去重判断：登录用户按user_id去重，游客按visitor_id去重；

3. 查询评论点赞点踩记录表，判断用户是否已操作过该评论：  
   
    - 已操作且类型相同：删除该记录，对应评论的点赞数/点踩数减1；
    
    - 已操作且类型不同：更新记录类型，原类型对应的计数减1，新类型对应的计数加1；
    
    - 未操作：插入新记录，对应评论的点赞数/点踩数加1；
    
4. 系统提交事务，返回最新的点赞数、点踩数，前端实时更新展示。

### 3.4.5 诗词纠错申请及审核修改流程

1. 普通用户（登录状态）进入诗词详情页，发现诗词内容（原文、释义、赏析等）存在错误，点击“纠错申请”按钮；

2. 用户填写纠错内容（明确错误位置、错误内容）、纠错依据（如权威文献、专业解读等），提交纠错申请；

3. 系统校验纠错申请内容合法性（长度、无敏感词），校验通过后，生成唯一纠错申请ID，记录申请信息，更新申请状态为“待审核”；

4. 系统通过消息推送组件，向所有具有资源审核角色的用户推送新纠错申请通知（包含申请ID、诗词标题、申请时间）；

5. 资源审核角色用户接收通知后，进入纠错审核页面，查看申请详情，进行投票（支持/驳回），填写投票意见（可选）；

6. 系统统计投票结果，按预设规则判断是否通过（如超过50%审核人员支持则通过，否则驳回）；

7. 若投票驳回：系统更新纠错申请状态为“已驳回”，推送消息通知申请人，告知驳回原因（结合审核人员意见）；

8. 若投票通过：系统更新纠错申请状态为“已通过”，推送消息通知所有具有资源修改角色的用户，分配修改任务（包含申请ID、诗词ID、纠错内容）；

9. 资源修改角色用户接收通知后，进入诗词修改页面，根据纠错申请内容修改对应诗词信息，修改完成后提交二次校验；

10. 校验通过：系统更新诗词数据，更新纠错申请状态为“已完成”，推送消息通知申请人；校验失败：退回修改任务，通知资源修改角色用户重新修改；

11. 申请人可在个人中心查看纠错申请的完整进度及最终结果。

## 3.5 非功能需求

1. 易用性：页面布局简洁直观，操作流程清晰，新手用户与游客可快速上手，无需复杂学习；纠错申请、审核、修改流程便捷，步骤明确。

2. 稳定性：支持高并发访问（同时在线1000+用户），服务无卡顿、无崩溃，接口响应时间≤1秒；纠错申请提交、消息推送、投票统计等操作实时响应。

3. 安全性：密码加密存储、登录权限拦截、敏感词过滤、防刷评论、游客隐私保护，防止数据泄露与恶意操作；纠错申请需验证用户身份，防止恶意纠错。

4. 可扩展性：模块独立设计，便于后期新增功能（如积分系统、打卡、排行榜、评论举报、纠错积分奖励）；

5. 可维护性：需求文档清晰、逻辑连贯，功能模块划分合理，便于后期迭代优化与需求调整；纠错流程可灵活调整投票规则、推送方式。

6. 兼容性：支持主流浏览器（Chrome、Edge、Firefox等），适配电脑、平板等不同终端访问。
